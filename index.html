Apresentando o Backstage

O portal unificado para acelerar o desenvolvimento no nosso laboratório.

Olá, equipe!

O Backstage é muito mais que um catálogo de software. É uma plataforma de código aberto, criada pelo Spotify, que funciona como um portal central para todos os nossos serviços, documentações, ferramentas e equipes. O objetivo é simples: organizar nosso ecossistema, reduzir a complexidade e permitir que todos nós possamos criar e entregar software de alta qualidade com mais velocidade e autonomia.

Principais Funcionalidades

Catálogo de Software

Uma fonte única de verdade para todos os nossos ativos: microserviços, bibliotecas, sites e modelos de ML. Saiba quem é o dono de quê, entenda as dependências e explore toda a nossa arquitetura em um só lugar.

TechDocs (Docs-as-Code)

Documentação técnica que vive junto com o código. Escrita em Markdown e versionada no Git, garantindo que a documentação esteja sempre atualizada, acessível e correta, diretamente na interface do Backstage.

Software Templates

Crie novos projetos em minutos, não em dias. Templates que automatizam a criação de repositórios, pipelines de CI/CD e toda a configuração inicial, garantindo as melhores práticas e padrões do laboratório desde o início.

Organização por Squads

Estrutura flexível que espelha nossa organização. Cada squad pode gerenciar seus próprios componentes e documentações, facilitando a autonomia e a colaboração entre as equipes.

Como o Backstage Transforma o Dia a Dia do Dev

Para o Dev Backend...

Precisa criar um novo microserviço? Use um template e tenha um projeto com CI/CD, testes e deploy configurados em 5 minutos. Precisa consumir uma API interna? Encontre a documentação, o dono e os endpoints no catálogo instantaneamente, sem precisar perguntar a ninguém.

Para o Dev Frontend...

Descubra facilmente quais serviços estão disponíveis para consumo, veja o status de cada um e acesse sua documentação de API em um só clique. Encontre e reutilize componentes de UI que já foram catalogados por outras equipes, promovendo consistência visual.

Para o DevOps...

Tenha uma visão completa de todos os serviços em produção. Use o catálogo para padronizar deployments, gerenciar configurações de infraestrutura (IaC) e associar documentação de runbooks diretamente aos serviços, agilizando a automação e a resposta a incidentes.

Para o Dev/SRE que está de plantão...

Um serviço está com problemas? Acesse o catálogo, identifique o componente, encontre o runbook de emergência (TechDocs) e veja os dashboards de monitoramento associados em segundos. A resposta a incidentes se torna muito mais rápida e eficiente.

Para o Novo Membro da Equipe...

Em vez de semanas de onboarding, um novo dev pode explorar o catálogo do Backstage para entender toda a arquitetura, descobrir quais serviços existem, quem são os responsáveis e como as coisas se conectam. A curva de aprendizado diminui drasticamente.

Requisitos para Ambiente Atual (Docker Compose)

  • Docker e Docker Compose instalados localmente.
  • Arquivo docker-compose.yml configurado com os serviços do Backstage (backend, frontend, database).
  • Arquivo de configuração app-config.yaml montado como um volume para gerenciar a configuração de forma externa.
  • Portas de rede (ex: 3000, 7007) devidamente mapeadas e disponíveis.

Requisitos para Futuro (Kubernetes)

  • Cluster Kubernetes acessível (Minikube, Kind, EKS, GKE, etc.).
  • Helm (ou Kustomize) para gerenciamento dos manifestos de deploy.
  • Configuração de Ingress para expor a aplicação com um hostname.
  • Gerenciamento de configurações e segredos via ConfigMaps e Secrets.
  • Solução de storage com Persistent Volumes para o banco de dados.
